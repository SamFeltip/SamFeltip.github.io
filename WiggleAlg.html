Overview

I designed an algorithm which would inspect thousands of nodes, find clusters of nodes, and adjust (or wiggle) them into a more suitable place, to make counting easier.

If the clusers contained more than 10 nodes, then a stamp would be placed ontop of them showing the number of nodes, rather than a pattern.

The problem

As shown on the three maps below, it is clear you cannot accurately count pins when they are too close together on a map.

Notice the blue circle on images 2 and 3. in the second image, the modes are so close together.

Also notice the red circle in images 1 and 2. In the first image, the map is so zoomed out, the 5 pins look like 1.

Nodes are all multicoloured to represent different types of pin

the nodes are too close together to count accurately

Notice the 5 cluser is now countable

Representing the maps

I implemented my algorithm on a model solution, and provided it as a function to be implemented into the mapping software by another team member.

I represented the problem on a smaller, more manipulateable scale.

This model is built using p5.js

Functional Algorithm: Drawing clusters vs stamps

If a small number of nodes are overlapping, a cluster is drawn in it's place.

If a eight or more clusters are overlapping, a stamp is drawn in their place.

Grayed samples are included under the pattern nodes to compare the initial vs transformed state

As the user zooms in and out, the clusters may change and patterns will change with them.

The Algorithm: Fitting Clusters

Not only are nodes overlapping, but it is important when the 'pattern' is drawn to replace the sample nodes, it resembles the initial nodes as closely as possible.

Pattern nodes can be rearranged to better fit the sample of map points (we call these sample nodes)

Each possible rearrangement in the model problem is given a name. Two possible rearrangements here are named 'GPBO' and 'POBG'. Each letter represents a colored pattern node.

A poor example of a pattern - lines are drawn between the pattern nodes (rings) and the sample nodes (white circles)

A better fit by rearranging the pattern nodes.

159 represents the sum of the distances between the pattern nodes and the sample nodes.

Notice the sum of the distances is much lower (72, rather than 159)

How the algorithm works: Pattern-sample relationships

This algorithm calculates the distance every sample node has in relation to every pattern node. These values give every sample node a level of 'relationship' to each pattern node. the lower the number, the stronger the relationship.

the sum of all the sample node relationships gives each sample node an 'eargency rating'. See below

A map displays disease outbreak with a heatmap

The alpha sample node has the following relationships: ADBC, where A is the closest and C is the furthest.

The alpha sample node has the following relationships: CBDA, where C is the closest and A is the furthest.

Eargency for the alpha sample node is 141. This node is in the centre of the pattern, so any node would make a reasonably good fit. This means the Eargency is low

Eargency for the gamma sample node is 239. This node is on the edge of the pattern, so it would only fit well to two pattern nodes. Therefore the eargency rating is higher than alpha.

How the algorithm works: Using pattern eargency to assign samples to patterns

Now we have all the information we need to decide two things:

First, we need to choose a sample to assign to a pattern.

Then, we need to decide which pattern to assign this sample too

We start with the sample with the highest sample eargency. We assign this to the closest available pattern

This is repeated until all n samples are assigned.
